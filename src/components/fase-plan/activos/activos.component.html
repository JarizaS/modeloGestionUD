8.1.2 Identificación de Activos de Información Críticos

Este modelo de gestión de riesgos se enfoca en los activos de información críticos de las PYMEs del sector T.I., incluyendo:

  • Procesos de desarrollo, operación y soporte tecnológico
  • Gestión de bases de datos y sistemas internos
  • Administración de redes, accesos y dispositivos conectados
  • Controles técnicos y organizativos de seguridad

Quedan excluidos procesos administrativos no tecnológicos, sistemas offline y certificaciones formales ISO.

Este modelo está diseñado como herramienta teórica adaptable para PYMEs con recursos limitados, bajo el enfoque del ciclo PDCA.

Los activos se clasifican por su naturaleza y función dentro de una PYME T.I., permitiendo estructurar el análisis de riesgos posterior.

Categoría Descripción del Activo Ejemplo Específico
Información Datos digitales con valor estratégico. Base de datos de clientes, contraseñas, logs.
Personas Usuarios internos con acceso a información sensible. Desarrolladores, soporte, administradores de red.
Software Aplicaciones para operación y gestión. CRM, ERP, Git, MySQL, herramientas SaaS.
Hardware Equipos físicos relacionados con TI. Servidores, portátiles, routers, switches.
Redes e Infraestructura Elementos de comunicación y conectividad. Firewalls, VPN, Wi-Fi, cableado estructurado.
Servicios de TI Servicios digitales internos o contratados. Correo corporativo, backups, hosting, nube.
Documentos Información estructurada y formalizada. Políticas, manuales, contratos, respaldos.

Se aplica el modelo CIA (Confidencialidad, Integridad, Disponibilidad) para determinar la criticidad de cada activo.

Categoría Activo Representativo Confidencialidad Integridad Disponibilidad Criticidad General
Información Base de datos de clientes Alta Alta Alta Crítico
Personas Soporte técnico Media Alta Alta Alta
Software Gestor de proyectos Alta Alta Media Crítico
Hardware Servidor de respaldo Media Alta Alta Crítico
Redes Firewall perimetral Alta Alta Alta Crítico
Servicios TI Correo corporativo Alta Media Alta Alta
Documentos Políticas internas Media Alta Media Alta

Criterios de valoración:

  • Alta: Impacto severo en operación, imagen o legalidad.
  • Media: Impacto importante pero no catastrófico.
  • Baja: Pérdida tolerable o fácilmente recuperable.

Criticidad General:

  • Crítico: Al menos 2 criterios en nivel Alto.
  • Alta: Uno en Alto, los demás Medios.
  • Moderada o baja: Ningún criterio en Alto.
